timeseries-qc

The open-source data quality-control layer for SCADA, DCS, IoT, and historian time-series data. Classify every row as Good, Suspect, or Bad, then render a multi-tag horizontal status timeline — the chart that no other open-source library produces.

$ pip install timeseries-qc

# Quickstart (5 lines)

import tsqc
import pandas as pd

df = pd.read_csv("sensor_data.csv")          # columns: timestamp, tag_name, value
result = tsqc.check(df, assume_tz="UTC")    # runs QC rules on every tag
result.plot().show()                            # multi-tag quality timeline
print(result.summary())                         # % good / sus / bad per tag

# Features

⚡

4 Built-in Rules

NullRule, FlatlineRule, DeltaRule, RangeRule — cover ≥80% of real-world bad data out of the box.

📊

Timeline Chart

Plotly Gantt-style chart: one row per tag, Green/Yellow/Red, interactive hover, range slider.

📄

YAML Config

Non-coders set thresholds in a text file. Glob-based tag matching supported.

🔧

Timestamp Health

Detect gaps, duplicates, non-monotonic timestamps, frequency drift, and DST ambiguity.

📦

HTML Export

Self-contained offline report with embedded Plotly chart, summary tables, and issue breakdown.

📈

Per-Issue Breakdown

Start/end times, row count, duration for every contiguous bad/sus segment.

Quality Labels

● good ● suspect ● bad

Each row is tagged with a quality label. When multiple rules fire, the worst level wins (bad > sus > good). Triggered rule names appear in quality_reasons.

# YAML Config Example

# tsqc_rules.yaml
default_rules:
  - check: null
    level: bad
  - check: flatline
    window: 1h
    min_delta: 0.001
    level: sus
  - check: delta
    threshold: 50.0
    level: sus

tag_rules:
  "FOREBAY.LEVEL":
    - check: range
      min: 900
      max: 1100
      level: bad
  "GENERATOR.*":
    - check: range
      min: 0
      max: 200
      level: bad
    - check: flatline
      window: 30min
      min_delta: 0.5
      level: sus
result = tsqc.check(df, rules="tsqc_rules.yaml")
result.summary()           # DataFrame: pct_good/sus/bad per tag
result.issue_summary()     # DataFrame: per-issue runs (start, end, rows, duration)
result.check_timestamps()  # DataFrame: gap/duplicate/non_monotonic issues
result.export_report("report.html")  # Full HTML with chart + all tables

# API Reference

tsqc.check(df, ...)

ArgumentDefaultDescription
time_col"timestamp"Name of the timestamp column.
tag_col"tag_name"Name of the tag column. None = single-tag mode.
value_col"value"Name of the value column.
rulesNoneList of Rule objects, path to a YAML file, or None (auto 3-sigma delta).
assume_tzNoneIANA timezone for tz-naive input (e.g. "UTC", "America/Chicago").

QCResult Methods

MethodReturnsDescription

.issue_summary()DataFrameContiguous bad/sus segments with start, end, rows, duration.
.check_timestamps()DataFrameGap, duplicate, non-monotonic, freq drift, DST issues.
.export_report(path)NoneSelf-contained HTML report (no CDN required).

# Known Limitations (v0.1.0)

  1. Pandas only. PySpark and Polars support are deferred.
  2. No YAML override of default rules. Tag-specific rules add to, not replace, default rules.
  3. Visualization requires Plotly ≥ 5.0. Matplotlib output not supported.
  4. DeltaRule is point-to-point diff only. Rolling-window delta is a v0.2 feature.
